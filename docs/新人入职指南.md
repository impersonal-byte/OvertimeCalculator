## 1. 项目概览

**加薪** 是一个纯原生的 Android 加班工资计算器应用，旨在为个人用户提供便捷的加班、调休记录以及工资预估功能。

* **业务痛点**：用户需要便捷地记录工作日、休息日、法定节假日的加班时长，并支持调休抵扣、自动按不同倍率计算当月加班工资，甚至支持通过已发工资反推当前时薪。
* **主要技术栈**：
  * **语言**：Kotlin (JDK 17)
  * **UI 框架**：Jetpack Compose、Material 3
  * **架构模式**：MVVM 架构 + 单向数据流 (UDF) (通过 `StateFlow` 与 Compose 结合)
  * **本地存储**：Room (用于关系型数据)、DataStore (用于用户偏好设置)
  * **后台任务**：WorkManager (用于静默同步节假日数据)
  * **依赖注入/容器**：手动依赖注入 (`AppContainer`)

## 2. 目录结构与分层架构

项目包名：`com.peter.overtimecalculator`，严格按照清晰的职责边界将代码分为三大层：`data`、`domain`、`ui`。

```text
app/src/main/java/com/peter/overtimecalculator/
├── data/       # 数据层：负责所有的数据存储、获取、网络请求与同步机制
│   ├── db/                 # Room 数据库定义 (Dao, Entities, TypeConverters)
│   ├── repository/         # 核心仓库类 (OvertimeRepository)，封装数据读写操作
│   ├── holiday/            # 节假日规则数据处理 (Timor API 解析, WorkManager 后台同步)
│   └── update/             # GitHub Release 版本检查更新模块
│
├── domain/     # 核心领域层：包含所有纯 Kotlin 的业务逻辑，不依赖 Android 框架
│   ├── Calculators.kt      # 核心计算器 (按倍率计算工资、调休抵扣逻辑、反推时薪算法)
│   ├── Models.kt           # 领域层业务模型实体 (DayType, MonthlyConfig 等)
│   ├── Validation.kt       # 各类规则校验
│   └── WriteUseCases.kt    # 将写入操作封装为 UseCase 给 ViewModel 调用
│
├── ui/         # 视图层：负责界面呈现、UI 状态管理和手势交互
│   ├── theme/              # 全局 UI 主题定义 (主题切换、动态取色、种子色)
│   ├── settings/           # 单独切分的设置页面 (DataManagement, Preferences 偏好设置等)
│   ├── Screens.kt          # Compose 页面基础导航和脚手架结构
│   └── OvertimeViewModel.kt# 维系整个应用 UI 状态的核心 ViewModel 
│
└── MainActivity.kt         # 应用的单一入口和宿主 Activity
└── OvertimeApplication.kt  # 全局 App 配置，初始化 AppContainer 与 WorkManager
```

## 3. 核心对象与组件解析

在新接触项目时，请重点关注以下几个类：

### 3.1 视图层 (UI)
* **`OvertimeViewModel`**:
  整个应用的核心 ViewModel。它通过 `AppUiState` 类暴露了整个当月的视图状态。它内部监听数据库变化 (`repository.observeMonth`)，组合当月配置、天数数据、用户外观设置等，通过 `StateFlow` 发射给 Compose 渲染。
* **单向数据流 (UDF)**:
  用户在界面上的操作 (如保存加班时长、修改当月时薪)，都会作为 `Intent` (事件) 传递给 `OvertimeViewModel`，由 ViewModel 调用 `UseCase` 写入数据库，数据库更新后再次驱动 Flow 刷新从而刷新界面。

### 3.2 领域层 (Domain)
领域层包含了项目的“灵魂”——加班工资是如何计算的。
* **`MonthlyOvertimeCalculator`**:
  计算指定月份下所有天的状态。其内部包含**调休抵扣规则**：调休时间优先抵扣工作日加班，其次休息日，最后是节假日。
* **`ReverseHourlyRateCalculator`**:
  提供特殊的“反推”功能。利用已发的加班工资，配合当月的加权时长（不同种类加班乘以对应倍率后的时长总计），计算出实际时薪。
* **`DayType` 枚举**:
  表示天的类型（工作日、休息日、节假日）。

### 3.3 数据层 (Data)
* **`AppDatabase`**:
  项目的 Room 数据库配置，由于使用了 `BigDecimal` 处理高精度货币问题，依赖了 `BigDecimalConverters`。核心表有三张：
  * `monthly_config`：每个月的配置 (时薪、倍率等)。
  * `overtime_entry`：记录用户每天登记的具体加班/调休时长。
  * `holiday_override`：用户手动覆盖的特殊天类型记录。
* **`HolidaySyncWorker`**:
  工作管理器。负责在后台通过 Timor API 下载中国的法定节假日数据并更新到本地，让系统能识别当年复杂的调休安排。

## 4. 关键业务流说明

### 4.1 日期类型判断优先级
在计算当天到底算哪种加班时，应用的判断优先级为：
1. **用户手动覆盖类型** (`overrideTypesByDate`)：如果用户人为指定今天是节假日，则最高优先。
2. **法定节假日规则** (`holidayCalendar`)：通过 Timor API 下载的网络数据。
3. **周末判断**：周六、周日默认归为休息日。
4. **工作日**：以上皆不满足，则认为是普通工作日。

### 4.2 加班金额的显示流
`数据库数据变更` -> `OvertimeRepository` (Flow) -> `OvertimeViewModel` -> 触发 `MonthlyOvertimeCalculator.calculate` (包含倍率相乘与 BigDecimal 精度格式化) -> `AppUiState` -> Compose 界面重组。

## 5. 开发调试与发版指引

### 5.1 环境要求
* 推荐使用 **Android Studio (最新稳定版)**
* **Agp (Android Gradle Plugin)** 结合 Kotlin Compose 插件。项目中使用了最新的 `org.jetbrains.kotlin.plugin.compose`。
* **Java 17** (必须)。

### 5.2 构建脚本与任务
我们在 `app/build.gradle.kts` 中定义了一些便捷任务：
* `packageReleaseApk`: 利用发版签名自动打包 release 并放入 `app/build/dist/` 下，文件名为 `OvertimeCalculator-<version>-universal.apk`。

### 5.3 GitHub Actions 流水线
项目已配置好了完整的 CI/CD：
* 提交或合入 `main` 分支时会触发并进行构建校验 (`Android CI`)。
* 推送版本 Tag (例如 `v1.6.0`) 时会触发 `Android Release` 工作流，由 GitHub 服务器自动签名并发布至 GitHub Releases 面板。*(签名证书放置在项目中并由 Secrets 提供密钥)*

### 5.4 最佳实践与规约
1. **KISS 原则**: 优先考虑简单清晰的实现方式，避免防御性过度工程。
2. **界面状态不变性**: 所有暴露给 UI 的 `State` 强制使用 `data class` 并避免可变属性 (var)。只通过 flow 传递全量新的状态树。
3. **精确的货币数值**: 涉及所有与金额、时薪相关的计算，全部在 Domain 与 Data 层使用 `BigDecimal` 类型处理（并且设置正确的格式化策略，例如 `RoundingMode.HALF_UP`），严禁使用 Double 或 Float 避免精度丢失。
